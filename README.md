# 自动定价项目

这个项目用于把采购单、尺寸重量表、竞品网页整理成统一输入草稿，并在人工确认后生成亚马逊美国站定价结果。

## 固定流程

1. 把原始文件放进 `input/`
2. 运行草稿脚本，生成 `output/产品名_统一输入草稿.xlsx`
3. 打开草稿，检查 SKU、尺寸重量、包数、竞品、手动定价
4. 运行正式定价脚本，生成 `output/产品名_正式定价结果.xlsx` 和报告

## 文件夹说明

- `input/purchase_orders/`：采购单 PDF 或 Excel
- `input/dimensions/`：尺寸重量表
- `input/competitors/`：竞品 HTML
- `input/rules/`：物流费、平台费、利润率规则
- `config/`：汇率、退货率、目标利润率等参数
- `scripts/`：自动化脚本
- `output/`：生成的草稿、定价结果和报告

## 第一步：生成统一输入草稿

在项目目录运行：

```bash
python3 scripts/generate_input_draft.py
```

生成：

```text
output/产品名_统一输入草稿.xlsx
```

你主要检查这些字段：

- SKU
- 变体/规格
- 销售包数
- 采购数量
- 采购单价
- 订单运费和优惠
- 包装长宽高
- 包装重量
- 头程运费
- 目标利润率
- 手动定价
- 竞品文件或链接

## 第二步：生成正式定价

草稿确认后运行：

```bash
python3 scripts/generate_final_pricing.py
```

生成：

```text
output/产品名_正式定价结果.xlsx
output/产品名_正式定价报告.md
```

## 重要规则

- 竞品和你的产品包数不同时，系统会按对比单位折算。
- 计费重量按 `max(实重, 体积重)` 计算。
- 采购单运费和优惠会按采购数量摊销。
- 如果草稿备注提示“由多个部件尺寸推断”，必须人工确认最终包装尺寸。
- 如果填写了手动定价，正式报告会按手动定价计算。

## 是否需要网页

当前阶段先使用文件夹流程。等流程稳定、需要多人使用或批量上传时，再做网页界面。
